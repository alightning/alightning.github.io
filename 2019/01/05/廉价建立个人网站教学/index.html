建立自己的个人网站（1）
网页
指导 2019-01-05 阅读

本文主要介绍如何低成本建立自己的个人网站，也算是对自己的网站做一下介绍吧。

前言

个人网站这种东西大家可能觉得挺陌生的，或者说觉得这个东西离自己比较遥远。但其实，我们可以用很短的时间和很低廉的成本建立一个自己的个人网站，本文就是对建立个人网站进行教学。

建立网站需要的条件

原则上，建立一个网站需要建站程序，服务器、域名

建站程序

常用的建站程序有很多，如wordpress,Z-blog等，本网站使用hexo博客框架，而本文也只介绍hexo框架下的个人博客网站的建立。

服务器

服务器分为虚拟主机、VPS、独立服务器等，服务器一般来说是收费的，但github上提供了’github page’这一可以将静态网页托管到github的网站服务，也就是说，我们不需要单独租服务器就可以让我们的网站正常运行。

域名

事实上，使用github托管网页就意味着你已经可以使用github.io进行登陆了，但github.io毕竟是一个二级域名，如果想拥有真正属于自己的域名，需要在网上申请一个，常用的申请网站由国内的万网和国外的godaddy。国内的域名价格高、而且需要备份，推荐godaddy这个网站，域名比较便宜的第一年只需要1美元，而且godaddy可以用支付宝付款。

本网站的二级域名是alightning.github.io

安装配置环境

首先，本教程基于windows操作系统，其他操作系统大同小异,查阅hexo官方文档下载对应软件。

建议所有用户都阅读一下hexo官方文档,有助于对hexo进行了解。
安装Node.js
安装git
下载页面是英文的，但我这样的英语菜鸟都会的大家也一定没问题。
全部安装完毕后，打开node.js，安装hexo:

sudo npm install hexo-cli -g

之后，我们需要建立博客文件夹，执行以下命令：

hexo init 
cd 
npm install

folder为你的文件夹名称
打开你建立的文件夹，应该可以找到以下目录：

.
├── _config.yml     //hexo配置文件
├── node_modules     //node.js运行库
├── package.json
├── scaffolds     //markdown模板
├── source     //资源文件夹
└── themes     //主题

输入指令检验是否配置完成：

hexo s

hexo s是本地调试指令，正常编译后会显示一个网址，用浏览器打开这个网址，我们应该会看到hexo给我们提供的初始界面helloword
到这里，网站的配置完成。
有任何不清楚的地方，可以观看官方视频进行学习。

布置你的博客

接下来，我们要将我们的博客布置到网络上。

github

首先，我们需要登陆github官网注册一个github账号，之后新建项目，注意项目名称一定要写用户名.github.io，每个账户只能建立一个github pages。建立完毕后我们需要把我们的public同步给github,我们通过安装插件的方法进行同步。
接下来，我们逐步进行操作：

  • 首先进入到博客文件夹, 安装hexo-deployer-git
cd 博客文件夹
npm install hexo-deployer-git --save

然后在博客的配置文件_config.yml（注意不是主题中的）中添加

deploy:
  type: git
  repo:
    github: 博客的git地址
    //gitcafe: 可以像这样部署到多个git服务器上
  • 设置github信息
git config --global user.name "你的用户名"    //注意加引号
git config --global user.email 你的邮箱    //不用加引号
  • 在本地生成ssh密钥
ssh-keygen -C 注册github所用的邮箱

这里的邮箱必须是在注册帐号时的邮箱, github和gitcafe的文档都是这么规定的. 如果两处的注册邮箱不同, 那就再生成一个ssh密钥呗, 注意避免重复命名覆盖文件~在命令运行的时候会有几处提示, Enter file in which to save the key指的是输入生成密钥的路径, 默认是~/.ssh, 也没必要改, 回车跳过；Enter passphrase是设置密码, 设置后每次使用这对密钥都会要求输入密码, 嫌麻烦就回车跳过。

在~/.ssh目录下有 id_rsa和 id_rsa.pub两个文件，其中id_rsa.pub文件里存放的即是公钥key。

登录到GitHub，点击右上方的头像，选择settings ，点击Add SSH key，把id_rsa.pub的内容复制到里面即可。

测试是否配置成功，输入代码：

ssh -T git@github.com

如果显示success，说明配置成功。

输入代码：

hexo g
hexo d

然后，在浏览器输入你的二级域名用户名.github.io便可访问你的网站。

同步域名

如果你不想购买一个属于自己的域名就已经可以满足，请跳过这一步

为了保证域名在国内的解析速度, 可以使用国内的DNS解析服务, 推荐DNSPod的DNS解析, 速度快, 而且免费的功能也完全足够用. 需要到域名注册商更换DNS服务器, 更换后需要过一段时间才会生效. 之后到dnspod的页面添加DNS记录, 添加cname记录, 记录值即为github pages给我们的二级域名。

打开你的github.io,进入settings，在github pages中找到custom domain,输入你的域名并点击save，这时你的github.io会生成一个cname文件，文件的内容是你的域名。将这个文件下载到你的电脑，并复制到你的博客文件夹/source,因为每次运行网页时都会删除之前的所有文件并将source中的文件转移至github.io。

现在，你可以使用hexo代码调试并布置你的博客：

hexo g -d      //等于hexo g,hexo d

CNAME解析需要一定的时间，正好我们也可以休息一下，今天的正文也就到这里了。
时不时可以输入你的域名试一下，CNAME解析时间大约在2个小时左右，也可能会更长。
最终，当你输入你的域名是，你可以看到你的网站，虽然现在还是只有hexo默认helloword，但想必你现在一定十分兴奋。

附录：常用hexo代码

常用的hexo命令:

$ hexo s    //相当于hexo server在本机预览网页
$ hexo g    //相当于hexo generate生成静态网页
$ hexo d    //相当于hexo deploy部署网页到服务器
$ hexo g -d    //hexo generate和hexo deploy的结合

以上命令已经足够你操作hexo了。
